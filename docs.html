Documentación de arquitectura y operación

Esta guía concentra la estructura interna de Chabi, cómo crear sesiones, cómo interactuar con la API y cómo entender los estados del sistema en producción.

Arquitectura

El flujo de Chabi está diseñado para conectar WhatsApp con un agente de IA persistente y desacoplado, manteniendo tanto la sesión como el historial bajo tu control.

WhatsApp Mobile -> WhatsappSocketEnvelope -> AgentWebSocketServer (8081) -> AiAgent
                  |                                  |
                  v                                  v
           ChatbotInitialSetup              ConversationStore
             (data/{bot}/)                    (data/{bot}/)

WebSocket

Chabi expone un servidor WebSocket dedicado para coordinar el agente, eventos de mensajería y sincronización de la sesión con el runtime del bot.

Host por defecto: ws://127.0.0.1:8081

Úsalo para instrumentación interna, conexiones de agentes y eventos en tiempo real.

Clases principales

WhatsappSocketEnvelope

Encapsula el socket de WhatsApp, coordina eventos entrantes y la recuperación del estado de sesión.

AiAgent / AiAgentBuilder

Construyen el agente, inyectan proveedor, modelo, prompt del sistema y herramientas disponibles.

AgentsMap

Mantiene el registro de agentes activos para una misma instancia de Chabi.

ConversationStore

Persiste memoria por conversación y permite restaurar contexto entre reinicios.

ChatbotInitialSetup

Prepara carpetas, archivos base y configuración inicial para cada nuevo chatbot en `data/{bot}/`.

Capa de persistencia

Chabi separa claramente persistencia de texto y persistencia estructurada para que puedas reconstruir contexto, estados y eventos sin perder trazabilidad.
